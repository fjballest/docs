MVC(1): move files

USAGE

        usage: mvc [-Dacmnv] {file} file
        -D: debug
        -a: preserve attributes
        -c: copy and remove to cross devices
        -m: behave like when moving many files
        -n: dry run; implies -v
        -v: report operations made

DESCRIPTION

Mvc moves files to a new destination. The last name given as an argument indicates the destination. If a single source file is moved then the destination is the exact name of the new file, and it is overwritten if it already exists.

If multiple files are being moved (or a single source with a predicate is used to find all matching files) then the destination for each file is the path resulting from appending to the destination specified in the command line the relative path of the source file with respect to the name indicated in the command line. That is, the move preserves the tree structure of the files being moved.

If the source and destinations are within the same device, a single rename request is issued and the move preserves all attributes.

If the move crosses a device boundary, it fails unless -c is given. In this case the file is copied and removed from the orginal location if the copy succeeds. Permissions are preserved in the copied file. Use -a to preserve all user attributes in this case.

EXAMPLES

Move the subtree rooted at the current directory to /tmp/dst: 

        ; mvc , /tmp/dst

Move all go source files and the directories to a similar tree at /tmp/dst: 

        ; mvc  ',~\.go$' /tmp/dst

Move file 1 to file 2 at /tmp: 

        ; mvc 1 /tmp/2

Move files 1 and 2 to /tmp: 

        ; mvc 1 2 /tmp
