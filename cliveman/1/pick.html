PICK(1): compile picky programs

USAGE

        usage: pick [-DELPSYdgv] [-o outfile] infile[..]

DESCRIPTION

pick compiles programs written in the Picky programming language. Picky is a programming language intended for a first course on computer programming. It is closer to a bicycle with two side-wheels than it is to a motor-bike. You won't probably use it unless you are using it for your first steps in programming.

The compiler generates byte-code, interpreted by an abstract machine for portability. The language is utterly picky, hence its name, by design.

These are the options:

    -D

      debug declarations

    -E

      debug expressions

    -L

      debug the lexer

    -P

      debug program

    -S

      dump statistics of resource usage (imprecise)

    -Y

      set Yydebug, for parser state debugging in yacc

    -d

      dump the stack on error before exiting

    -g

      enable global variables, forbidden by default

    -o out.pam

      output file name

    -v

      return version of the build of the compiler, the number is seconds since epoch when the build happened.

SOURCE

BUGS

Statistics should be more precise.

REFERENCES
