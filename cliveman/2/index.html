Section 2: go packages

  • intro: introduction to clive packages
  • ch: Channels that can go through external I/O devices for Clive.
  • cmd: Clive command tools and interfaces
  • cmd.child: Report the set of pids for a command and all its descendants
  • cmd.look: Return commands to run when the user looks for strings.
  • cmd.opt: command line options, in the unix style and working on any argv[] array.
  • cmd.run: run a command and use channels for I/O
  • cmd.test: Help for testing commands
  • cmd.tty: UNIX tty tools
  • cmd.wr.frmt: CONSTANTS
  • cmd.wr.refs: References for wr using the refer format
  • dbg: Debug tools
  • fswatch: file system watcher
  • fuse: FUSE service loop, from bazil.org.
  • fuse.ostest: Utilities to aid in tests of fuse file systems. Mimics zx/fstest but using
  • mblk: Memory block buffers
  • mblk.rwtest: Test tools for things that can be read and written.
  • net: Network services and tools over clive/ch chans.
  • net.auth: Authentication services for clive.
  • net.ink: Clive's ink: web based user interfaces.
  • net.ink.js: VARIABLES
  • ns: name spaces.
  • snarf: Access to the system clipboard.
  • sre: Structural Regular Expressions on rune based text.
  • txt: In-memory text with insertion, removal, selection, and marks
  • u: users
  • u.user: User events. Currently works only for osx but it should be trivial to fix it
  • work: A pool of worker processes.
  • zx: Clive ZX file system tools.
  • zx.fscmp: Compare two file systems. For testing mostly and not using find. Refer to
  • zx.fstest: Utilities to aid in tests of zx file systems packages
  • zx.pred: Predicates on files as used by the zx interfaces
  • zx.repl: Tools to synchronize zx file trees
  • zx.rzx: Remote ZX access
  • zx.zux: ZX for UX file systems
  • zx.zxc: ZX caching fs
  • zx.zxfs: FUSE server for zx.
