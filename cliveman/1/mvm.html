MVM(1): move multiple files

USAGE

        usage: mvm [-Dcnv] [dst]
        -D: debug
        -c: collect all files into the destination dir
        -n: dry run
        -v: report operations made

DESCRIPTION

Mvm moves multiple files (or directories) to a target directory. The argument indicates the destination directory and defaults to the current directory.

The destination for each file is the path resulting from appending to the destination path the relative path of the source file with respect to the name indicated in the command line. In particular:

  • Moving /a/dir to a destination moves the tree at /a/dir to the tree rooted at the destination.
  • Moving afile to a destination moves the file to a file with the same name in the destination directory.

That is, the move preserves the tree structure of the files being moved. Under flag -c all files are moved directly into the destination directory, including directories.

The program refuses to move files if the operations crosses a device. If the caller cares, it can rely on cpm(1) and rem(1) to achieve the same effect.

EXAMPLES

Move the subtree rooted at the current directory to /tmp/dst: 

        ; lf ,| mvm /tmp/dst

Move all go source files from /tmp/src at the current directory: 

        ; lf  '~\.go$'  | mvm

Move files 1 and 2 to /tmp: 

        ; lf 1 2 | mvm /tmp

Move file 1 at /tmp/1: 

        ; lf 1  | mvm /tmp

SOURCE

  • /zx/sys/src/clive/app/mvm

SEE ALSO

  • lf(1) to select files to move.
  • mvf(1) for a related command that moves a single file.
  • cpm(1) to copy files.
  • rem(1) to remove files.
