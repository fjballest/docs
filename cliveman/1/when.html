WHEN(1): notify the user or run commands when files change

USAGE

        usage: when [-1Du] file [cmd...]
        -1: terminate after the first change
        -D: debug
        -u: don't use unix out

DESCRIPTION

When watches for file system changes and prints the changed file names or executes a command, if given. If the file is a directory it is considered as changed when either the directory or its contents change. But there is no recursion to sub-directories.

If the cmd given contains a "%" sign it is replaced with the name for the changed file.

EXAMPLES

    Useful to run when editing sources: 
        ; when . Inst -v

    Or perhaps 

        ; when . Lgo test -v
