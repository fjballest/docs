CPF(1): copy a single file

USAGE

        usage: cpf [-Dacnv] [dst]
        -D: debug
        -a: preserve attributes
        -c: collect all files into the destination dir
        -n: dry run
        -v: report operations made

DESCRIPTION

Cpf copies a single file (or directory) to a target name. The destination is the exact name of the new copy for the file, and it is overwritten if it already exists.

This command is the same binary for cpm(1) and thus the destination is optional, but it is required for cpf. Also, flag -c is meaningless when used here.

To avoid confussion regarding what happens when a single source and a single destination is given, depending on the existence of the destination, Clive has another command, cpm(1), to copy multiple files into a destination directory. For cpf, the desitnation is always the exact file name for the copied source.

EXAMPLES

Copy file 1 to file 2 at /tmp: 

        ; gf 1 | cpf /tmp/2

Note that this is an error, because the source is a file and the copy name is a directory. 

        ; gf 1 | cpf /tmp

SOURCE

  • /zx/sys/src/clive/app/cpm

SEE ALSO

  • gf(1) to select files to copy.
  • cpm(1) for a related command that copies multiple files.
