CPM(1): copy multiple files

USAGE

        usage: cpm [-Dacnv] [dst]
        -D: debug
        -a: preserve attributes
        -c: collect all files into the destination dir
        -n: dry run
        -v: report operations made

DESCRIPTION

Cpm copies multiple files (or directories) to a target directory. The argument indicates the destination directory and defaults to the current directory.

The destination for each file is the path resulting from appending to the destination path the relative path of the source file with respect to the name indicated in the command line. In particular:

  • Copying ., to a destination copies the tree at . to the tree rooted at the destination.
  • Copying afile to a destination copies the file to a file with the same name in the destination directory.

That is, the copy preserves the tree structure of the files being copied. Under flag -c all files are copied directly at the destination directory, including directories.

Only file permissions are preserved in the copy. Use -a to preserve all user attributes.

EXAMPLES

Copy the subtree rooted at the current directory to /tmp/dst: 

        ; gf ,| cpm /tmp/dst

Copy all go source files and the directories from /tmp/src at the current directory: 

        ; gf  '/tmp/src,d|~\.go$'  | cpm

Copy files 1 and 2 to /tmp: 

        ; gf 1 2 | cpm /tmp

Copy file 1 at /tmp/1: 

        ; gf 1  | cpm /tmp

SOURCE

  • /zx/sys/src/clive/app/cpm

SEE ALSO

  • gf(1) to get files to copy.
  • cpf(1) for a related command that copies a single file.
