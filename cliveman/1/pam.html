PAM(1): Picky Abstract Machine interpreter

USAGE

        usage: pam [-CDEFIMSXds] [-w] file

DESCRIPTION

    pam interprets acts as an interpreter for the Picky Abstract Machine which pick(1) compiles to.

Multimedia is served as a web page using javascript. In Unix-like environments pam is called automatically using the [verb #!/bin/pam ] prelude. In Windows the .pam extension is registered for the same purpose.

These are the options:

    -C

      cpu profiling

    -D

      memory allocation debug

    -E

      memory profiling

    -F

      binary file debug

    -I

      file i/o debug

    -M

      globals debug

    -S

      stack debug

    -X

      instruction execution debug

    -d

      dump stack on error before exiting

    -s

      dump statistics

    -w

      don't wait at the end in Windows for when it is being run inside an environment like SciTE

SOURCE

  • /zx/sys/src/clive/cmd/picky

BUGS

Statistics should be more precise.

REFERENCES
